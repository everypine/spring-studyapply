## 1. 서비스 개요

사용자는 스터디를 개설할 수 있고, 다른 사용자는 모집 중인 스터디에 참가 신청할 수 있다.

스터디장은 들어온 신청을 승인하거나 거절할 수 있으며, 승인된 인원은 스터디 정원을 초과할 수 없다.

## 2. 주요 기능
| 기능         | 설명                              |
|------------|---------------------------------|
| 회원 생성      | 사용자는 서비스 이용을 위해 회원을 생성할 수 있다    |
| 스터디 생성     | 회원은 새로운 스터디를 개설할 수 있다           |
| 스터디 목록 조회  | 사용자는 스터디 목록을 조회할 수 있다           |
| 스터디 상세 조회  | 사용자는 특정 스터디의 상세 정보를 조회할 수 있다    |
| 스터디 신청     | 사용자는 모집 중인 스터디에 참가 신청할 수 있다     |
| 내 신청 목록 조회 | 사용자는 자신이 신청한 스터디 목록을 조회할 수 있다   |
| 신청 취소      | 사용자는 본인이 제출한 신청을 취소할 수 있다       |
| 신청 승인      | 스터디장은 자신의 스터디에 들어온 신청을 승인할 수 있다 |
| 신청 거절      | 스터디장은 자신의 스터디에 들어온 신청을 거절할 수 있다 |
| 모집 종료      | 스터디장은 자신의 스터디 모집을 종료할 수 있다      |

## 3. 도메인 규칙

### 회원
- 회원은 이름, 이메일, 비밀번호를 가진다. 
- 이메일은 중복될 수 없다. 
- 회원은 여러 개의 스터디를 개설할 수 있다. 
- 회원은 여러 스터디에 참가 신청할 수 있다.

### 스터디
- 스터디는 제목, 설명, 정원을 가진다. 
- 스터디를 생성한 회원은 해당 스터디의 스터디장이 된다. 
- 스터디 정원은 1명 이상이어야 한다. 
- 스터디는 모집 중 또는 모집 종료 상태를 가진다. 
- 모집 종료된 스터디에는 새로 신청할 수 없다.

### 스터디 신청
- 회원은 모집 중인 스터디에 참가 신청할 수 있다. 
- 같은 회원은 같은 스터디에 중복 신청할 수 없다. 
- 스터디장은 자신이 만든 스터디에 신청할 수 없다. 
- 신청은 처음 생성될 때 승인 대기 상태가 된다. 
- 신청자는 승인 대기 상태의 신청만 취소할 수 있다. 
- 스터디장은 승인 대기 상태의 신청만 승인하거나 거절할 수 있다. 
- 승인된 신청 수는 스터디 정원을 초과할 수 없다.

## 4. 상태 정의
### 스터디 상태
   | 상태    | 설명                   |
   |-------|----------------------|
   | 모집 중  | 사용자가 참가 신청할 수 있는 상태  |
   | 모집 종료 | 더 이상 참가 신청을 받지 않는 상태 |

### 신청 상태
   | 상태    | 설명                      |
   |-------|-------------------------|
   | 승인 대기 | 사용자가 신청했고 아직 처리되지 않은 상태 |
   | 승인됨   | 스터디장이 신청을 승인한 상태        |
   | 거절됨   | 스터디장이 신청을 거절한 상태        |
   | 취소됨   | 신청자가 직접 신청을 취소한 상태      |

## 5. 권한 규칙
| 기능     | 허용 사용자       |
|--------|--------------|
| 스터디 생성 | 회원           |
| 스터디 신청 | 스터디장이 아닌 회원  |
| 신청 취소  | 신청자 본인       |
| 신청 승인  | 해당 스터디의 스터디장 |
| 신청 거절  | 해당 스터디의 스터디장 |
| 모집 종료  | 해당 스터디의 스터디장 |

## 6. 예외 상황

다음 상황에서는 요청이 실패해야 한다.

- 존재하지 않는 회원이 요청한 경우 
- 존재하지 않는 스터디를 조회하거나 신청한 경우 
- 존재하지 않는 신청을 처리하려는 경우 
- 이미 같은 스터디에 신청한 경우 
- 스터디장이 자신의 스터디에 신청한 경우 
- 모집 종료된 스터디에 신청한 경우 
- 정원이 가득 찬 스터디에 신청한 경우 
- 권한이 없는 사용자가 신청 승인, 거절, 취소, 모집 종료를 시도한 경우 
- 이미 처리된 신청을 다시 승인, 거절, 취소하려는 경우

## 7. 비즈니스 제약 조건
| 구분    | 제약                             |
|-------|--------------------------------|
| 회원    | 이메일은 중복될 수 없다                  |
| 스터디   | 정원은 1명 이상이어야 한다                |
| 스터디   | 모집 종료 상태에서는 새 신청을 받을 수 없다      |
| 신청    | 같은 회원은 같은 스터디에 중복 신청할 수 없다     |
| 신청    | 스터디장은 자신의 스터디에 신청할 수 없다        |
| 신청    | 승인된 신청 수는 스터디 정원을 초과할 수 없다     |
| 신청 처리 | 승인 대기 상태의 신청만 승인, 거절, 취소할 수 있다 |
